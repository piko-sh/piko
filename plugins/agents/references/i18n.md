# Internationalisation (i18n)

Use this guide when adding multi-language support, translations, or locale routing.

## i18n block

Define translations in the `<i18n>` section of a `.pk` file:

```piko
<i18n lang="json">
{
  "en": {
    "greeting": "Hello",
    "nav": {
      "home": "Home",
      "about": "About"
    }
  },
  "fr": {
    "greeting": "Bonjour",
    "nav": {
      "home": "Accueil",
      "about": "À propos"
    }
  }
}
</i18n>
```

Multiple `<i18n>` blocks per file are allowed (they merge).

## Accessing translations

### In templates

```piko
<!-- Local (component-scoped) translation -->
<h1>{{ LT("greeting") }}</h1>
<nav>{{ LT("nav.home") }}</nav>

<!-- Global translation (from global i18n files) -->
<p>{{ T("common.footer") }}</p>
```

### In Go code

```go
func Render(r *piko.RequestData, props piko.NoProps) (Response, piko.Metadata, error) {
    greeting := r.LT("greeting", "Hello")  // With fallback
    footer := r.T("common.footer")          // Global lookup
    return Response{Greeting: greeting}, piko.Metadata{}, nil
}
```

## Translation functions

| Function | Scope | Description |
|----------|-------|-------------|
| `T(key, fallback...)` | Global + local | Looks up from global translation files (can also access local translations) |
| `LT(key, fallback...)` | Component | Looks up specifically from the component's `<i18n>` block |

Both accept optional fallback values. If no translation is found and no fallback is provided, the key itself is returned.

## String variables

Inject dynamic values into translations:

```json
{
  "en": {
    "welcome": "Welcome, {{name}}!"
  }
}
```

```go
msg := r.LT("welcome", piko.StringVar("name", user.Name))
```

## Pluralisation

Use pipe-separated forms:

```json
{
  "en": {
    "items": "{{count}} item|{{count}} items"
  }
}
```

```go
msg := r.LT("items", piko.StringVar("count", strconv.Itoa(count)))
```

The framework selects the correct form based on the count value.

## Locale routing strategies

Configure in `piko.yaml`:

### prefix

All locales get a URL prefix:

```text
/en/about
/fr/about
```

### prefix_except_default

Default locale has no prefix, others do:

```text
/about       (default locale, e.g. en)
/fr/about    (non-default)
```

## Current locale

Access in Go:

```go
locale := r.Locale()         // Current request locale
defaultLocale := r.DefaultLocale()  // Fallback locale
```

## Supported locales

Declare per-component:

```go
func SupportedLocales() []string {
    return []string{"en", "fr", "de"}
}
```

## Locale-aware links

Use `<piko:a>` for automatic locale-prefixed links:

```piko
<piko:a href="/about">About</piko:a>
<!-- Renders as /fr/about when locale is fr -->
```

## Fallback chain

When a translation key is missing:
1. Look in the requested locale (local `<i18n>` block)
2. Fall back to the default locale (local `<i18n>` block)
3. Look in global translation files
4. Use the provided fallback argument
5. Return the raw key

## LLM mistake checklist

- Using `T()` when you mean `LT()` (component-scoped translations need `LT`)
- Forgetting `lang="json"` on the `<i18n>` tag
- Using single braces `{name}` instead of double `{{name}}` for string variables
- Hardcoding locale in URLs instead of using `<piko:a>`
- Forgetting to declare `SupportedLocales()` when the component has translations
