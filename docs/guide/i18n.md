# Internationalisation (i18n)

Piko provides multi-language support with locale-aware routing, automatic fallback translations, and SEO optimisation.

## Overview

Piko's i18n system supports:

- **Translation files** - JSON files with nested or flat keys
- **Locale routing** - URL-based locale detection with configurable strategies
- **Fallback chains** - Automatic fallback from `en-GB` to `en` to default locale
- **Component-scoped translations** - Define translations within `<i18n>` blocks
- **SEO metadata** - Generate hreflang tags for multi-language pages

## Quick start

### 1. Create translation files

Create JSON files in the `i18n/` directory, named by locale:

```text
myapp/
├── i18n/
│   ├── en.json
│   ├── fr.json
│   └── de.json
├── pages/
└── config.json
```

Each file contains translations as key-value pairs:

```json
{
  "greeting": "Hello",
  "farewell": "Goodbye",
  "user": {
    "welcome": "Welcome, ${name}!"
  }
}
```

Nested keys are flattened to dot notation. The above creates keys `greeting`, `farewell`, and `user.welcome`.

### 2. Configure i18n

Add the `i18n` section to your `config.json`:

```json
{
  "name": "My Website",
  "i18n": {
    "defaultLocale": "en",
    "strategy": "prefix_except_default",
    "locales": ["en", "fr", "de"]
  }
}
```

**Configuration options:**

| Field | Description |
|-------|-------------|
| `defaultLocale` | Fallback locale when no match is found |
| `strategy` | URL routing strategy (`prefix` or `prefix_except_default`) |
| `locales` | List of supported locale codes |

### 3. Use translations

Call `T()` in your templates to fetch translations:

```html
<template>
  <h1>{{ T("greeting") }}</h1>
  <p>{{ T("user.welcome").StringVar("name", state.UserName) }}</p>
</template>

<script type="application/x-go">
package main

import "piko.sh/piko"

type Response struct {
    UserName string
}

func Render(r *piko.RequestData, props piko.NoProps) (Response, piko.Metadata, error) {
    return Response{UserName: "Alice"}, piko.Metadata{}, nil
}
</script>
```

## Translation files

### File structure

Translation files use JSON with nested or flat structure:

**Nested structure:**

```json
{
  "nav": {
    "home": "Home",
    "about": "About Us"
  },
  "errors": {
    "not_found": "Page not found"
  }
}
```

**Flat structure:**

```json
{
  "nav.home": "Home",
  "nav.about": "About Us",
  "errors.not_found": "Page not found"
}
```

Both structures are equivalent. Piko flattens nested keys to dot notation internally.

### File naming

Name translation files by their locale code:

- `en.json` - English (generic)
- `en-GB.json` - British English
- `en-US.json` - American English
- `fr.json` - French
- `de.json` - German

## Routing strategies

Piko supports two URL routing strategies for multi-language sites.

### `prefix`

All locales include the locale code in the URL:

| Locale | URL |
|--------|-----|
| English | `/en/about` |
| French | `/fr/about` |
| German | `/de/about` |

### `prefix_except_default`

The default locale has no prefix; other locales are prefixed:

| Locale | URL |
|--------|-----|
| English (default) | `/about` |
| French | `/fr/about` |
| German | `/de/about` |

This is the recommended strategy for most sites.

## Locale detection

Piko determines the current locale from:

1. **URL path** - The locale prefix in the URL (e.g., `/fr/about`)
2. **Fallback** - Uses the configured `defaultLocale`

The current locale is available via `r.Locale()` in your Render function.

## Fallback chains

When a translation key is missing, Piko searches in order:

1. **Exact locale** - e.g., `en-GB`
2. **Base language** - e.g., `en`
3. **Default locale** - Configured `defaultLocale`

For example, if looking up `greeting` in locale `en-GB`:

1. Check `en-GB.json` for `greeting`
2. Check `en.json` for `greeting`
3. Check the default locale file

If no translation is found, the key itself is returned.

## Component-scoped translations

Define translations within a component using the `<i18n>` block:

```html
<template>
  <button>{{ LT("save") }}</button>
</template>

<i18n lang="json">
{
  "en": {
    "save": "Save"
  },
  "fr": {
    "save": "Enregistrer"
  }
}
</i18n>
```

Use `LT()` (local translation) to access component-scoped translations. This is useful for translations that are only relevant to a single component.

### T() vs LT()

| Function | Searches | Use case |
|----------|----------|----------|
| `T()` | Global files first, then local | Shared translations across the app |
| `LT()` | Local `<i18n>` block only | Component-specific translations |

## SEO metadata

Generate hreflang tags for multi-language SEO using `GenerateLocaleHead`:

```go
func Render(r *piko.RequestData, props piko.NoProps) (Response, piko.Metadata, error) {
    i18nConfig := piko.I18nConfig{
        DefaultLocale: "en",
        Strategy:      "prefix_except_default",
        Locales:       []string{"en", "fr", "de"},
    }

    locale, canonical, alternates := piko.GenerateLocaleHead(
        r, i18nConfig, "/about", nil,
    )

    metadata := piko.Metadata{
        Language:       locale,
        CanonicalURL:   canonical,
        AlternateLinks: alternates,
    }

    return Response{}, metadata, nil
}
```

This generates:

- **Language** - Current page locale
- **Canonical URL** - URL of the default locale version
- **Alternate links** - hreflang links for all locales plus x-default

### Override supported locales

For pages that don't support all configured locales, pass an override:

```go
// This page only has English and German versions
supportedHere := []string{"en", "de"}
locale, canonical, alternates := piko.GenerateLocaleHead(
    r, i18nConfig, "/special-page", supportedHere,
)
```

## Variable interpolation

Embed dynamic values in translations using `${expression}` syntax:

**Translation file:**

```json
{
  "greeting": "Hello, ${name}!",
  "items_count": "You have ${count} items"
}
```

**Usage:**

```go
T("greeting").StringVar("name", "Alice")  // "Hello, Alice!"
T("items_count").IntVar("count", 5)       // "You have 5 items"
```

See [i18n Advanced](/docs/guide/i18n-advanced) for all variable types including `FloatVar`, `DecimalVar`, `MoneyVar`, `BigIntVar`, `TimeVar`, and `DateTimeVar`.

## Pluralisation

Define plural forms separated by pipes (`|`):

**Translation file:**

```json
{
  "items": "one item|${count} items"
}
```

**Usage:**

```go
T("items").Count(1)   // "one item"
T("items").Count(5)   // "5 items"
```

Piko implements CLDR plural rules for accurate pluralisation across languages. See [i18n Advanced](/docs/guide/i18n-advanced) for language-specific plural forms.

## Complete example

```html
<template>
  <div>
    <h1>{{ T("page.title") }}</h1>
    <p>{{ T("page.welcome").StringVar("name", state.User.Name) }}</p>
    <p>{{ T("page.items").Count(state.ItemCount) }}</p>

    <!-- Component-scoped translation -->
    <button>{{ LT("submit") }}</button>
  </div>
</template>

<script type="application/x-go">
package main

import "piko.sh/piko"

type User struct {
    Name string
}

type Response struct {
    User      User
    ItemCount int
}

func Render(r *piko.RequestData, props piko.NoProps) (Response, piko.Metadata, error) {
    i18nConfig := piko.I18nConfig{
        DefaultLocale: "en",
        Strategy:      "prefix_except_default",
        Locales:       []string{"en", "fr"},
    }

    locale, canonical, alternates := piko.GenerateLocaleHead(
        r, i18nConfig, "/dashboard", nil,
    )

    return Response{
        User:      User{Name: "Alice"},
        ItemCount: 3,
    }, piko.Metadata{
        Language:       locale,
        CanonicalURL:   canonical,
        AlternateLinks: alternates,
    }, nil
}
</script>

<i18n lang="json">
{
  "en": {
    "submit": "Submit"
  },
  "fr": {
    "submit": "Soumettre"
  }
}
</i18n>
```

With `i18n/en.json`:

```json
{
  "page": {
    "title": "Dashboard",
    "welcome": "Welcome back, ${name}!",
    "items": "one item in your cart|${count} items in your cart"
  }
}
```

And `i18n/fr.json`:

```json
{
  "page": {
    "title": "Tableau de bord",
    "welcome": "Bienvenue, ${name} !",
    "items": "un article dans votre panier|${count} articles dans votre panier"
  }
}
```
